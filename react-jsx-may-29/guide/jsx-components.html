Converted JSX ‚öõÔ∏è

JSX
// App.jsx
import './App.css';

function App() {
  const showMore = () => {
    alert('More info coming soon!');
  };

  return (
    <div className="App">
      <div className="header">
        <h1>John Doe</h1>
        <p>Web Developer</p>
      </div>
      <div className="card">
        <h2>About Me</h2>
        <p>I love coding!</p>
        <button onClick={showMore}>Learn More</button>
      </div>
    </div>
  );
}

export default App;

üîß Key Conversion Rules

1. class ‚Üí className
HTML: <div class="header">
JSX: <div className="header">
2. onclick ‚Üí onClick (camelCase)
HTML: <button onclick="showMore()">
JSX: <button onClick={showMore}>
3. Self-closing tags need closing slash
HTML: <img src="photo.jpg">
JSX: <img src="photo.jpg" />
4. JavaScript expressions in curly braces
HTML: <p>Static text</p>
JSX: <p>{dynamicVariable}</p>

Step 2: CSS Integration Methods üé®

React gives you multiple ways to style your components. Each method has its own superpowers!

üóÇÔ∏è CSS Files (Traditional)

Best for: Global styles, beginners

CSS File

  padding: 2rem;
  text-align: center;
}
JSX

  return <div className="header">Hello!</div>;

‚úÖ Pros: Familiar, easy to learn, good for global styles
‚ö†Ô∏è Cons: Global scope can cause conflicts

üìù Inline Styles (JavaScript Objects)

Best for: Dynamic styles, component-specific styling

JSX
function Header({ isActive }) {
  const headerStyle = {
    background: isActive ? '#4caf50' : '#f44336',
    color: 'white',
    padding: '2rem',
    textAlign: 'center', // camelCase!
    borderRadius: '8px'
  };

  return <div style={headerStyle}>Dynamic Header</div>;
}
‚úÖ Pros: Dynamic, component-scoped, no conflicts
‚ö†Ô∏è Cons: No pseudo-classes, limited features

üè† CSS Modules (Scoped Styles)

Best for: Component-specific styles with full CSS features

Header.module.css
.header {
  background: linear-gradient(45deg, #667eea, #764ba2);
  color: white;
  padding: 2rem;
}

.header:hover {
  transform: scale(1.05);
  transition: transform 0.3s ease;
}
Header.jsx
import styles from './Header.module.css';

function Header() {
  return (
    <div className={styles.header}>
      Scoped Header
    </div>
  );
}
‚úÖ Pros: Scoped styles, full CSS features, no conflicts
‚ö†Ô∏è Cons: Extra setup, class name references

üíÖ Styled Components (CSS-in-JS)

Best for: Dynamic, themed applications

Terminal
npm install styled-components

‚úÖ Pros: Dynamic styling, theming, component-scoped
‚ö†Ô∏è Cons: Learning curve, bundle size

Step 3: Component Breakdown Strategy üß©

üçï Componentization Analogy: Pizza Restaurant

Think of your webpage like a pizza restaurant. Instead of making every pizza from scratch each time, you create reusable "recipes" (components):
‚Ä¢ Dough Component: Base that every pizza uses
‚Ä¢ Sauce Component: Can be tomato, white, or BBQ
‚Ä¢ Topping Component: Reusable pepperoni, cheese, vegetables
‚Ä¢ Pizza Component: Combines all ingredients based on order

üå≥ Component Tree Structure

Breaking Down Our Portfolio Example

üéØ Step-by-Step Componentization

1. Header Component
// components/Header.jsx
function Header({ name, role }) {
  return (
    <header className="header">
      <h1>{name}</h1>
      <p>{role}</p>
    </header>
  );
}

export default Header;
2. Card Component
// components/Card.jsx
function Card({ title, content, onButtonClick }) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <p>{content}</p>
      <button onClick={onButtonClick}>
        Learn More
      </button>
    </div>
  );
}

export default Card;
3. Main App Component
// App.jsx
import Header from './components/Header';
import Card from './components/Card';
import './App.css';

function App() {
  const handleShowMore = () => {
    alert('More info coming soon!');
  };

  return (
    <div className="App">
      <Header name="John Doe" role="Web Developer" />
      <Card
        title="About Me"
        content="I love coding!"
        onButtonClick={handleShowMore}
      />
    </div>
  );
}

export default App;

Step 4: Module Import/Export System üì¶

Understanding ES6 Modules

ES6 modules are like a library system - you can "check out" (import) code from other files and "lend" (export) your code to others!

Export Methods üì§

Default Export (One per file)

Button.jsx
function Button({ text, onClick }) {
  return (
    <button onClick={onClick}>
      {text}
    </button>
  );
}

export default Button;
App.jsx
import Button from './Button';
// OR import MyButton from './Button'; (any name works)

Named Export (Multiple per file)

utils.js
export const formatDate = (date) => {
  return date.toLocaleDateString();
};

export const capitalizeText = (text) => {
  return text.charAt(0).toUpperCase() + text.slice(1);
};

const helper = () => console.log('helper');
export { helper };
App.jsx
import { formatDate, capitalizeText } from './utils';
// OR import * as utils from './utils';
// Then use: utils.formatDate()
