Part 1: Understanding useState Hook üì¶

What is useState? ü§î

useState is a Hook that allows you to add state to functional components. Before hooks, only class components could have state. Now, functional components can remember values between renders!

‚ùå Before Hooks (Class Components)

class Counter extends React.Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  increment = () => {
    this.setState({ count: this.state.count + 1 });
  }

  render() {
    return (
      <div>
        <p>Count: {this.state.count}</p>
        <button onClick={this.increment}>
          +1
        </button>
      </div>
    );
  }
}

Problems:

  • Lots of boilerplate code
  • Confusing 'this' binding
  • Constructor setup required
  • Harder to test and reuse logic

‚úÖ With useState Hook

import React, { useState } from 'react';

function Counter() {
  const [count, setCount] = useState(0);

  const increment = () => {
    setCount(count + 1);
  };

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={increment}>
        +1
      </button>
    </div>
  );
}

Benefits:

  • Much cleaner and shorter
  • No 'this' confusion
  • Easy to understand
  • Reusable and testable
üè† useState Like a Smart Home System

Think of useState like a smart home system:
‚Ä¢ State variable (count) = current temperature display
‚Ä¢ Setter function (setCount) = thermostat control
‚Ä¢ Re-render = display updates when temperature changes
‚Ä¢ Just like a thermostat remembers and displays the current temperature, useState remembers and displays your data!

useState Deep Dive - Advanced Patterns üîç

Different Types of State

useState can handle any type of data - strings, numbers, booleans, objects, arrays, and more!

import React, { useState } from 'react';

function StateExamples() {
  // Primitive values
  const [name, setName] = useState('John');
  const [age, setAge] = useState(25);
  const [isVisible, setIsVisible] = useState(true);

  // Objects
  const [user, setUser] = useState({
    name: 'Alice',
    email: 'alice@example.com',
    role: 'admin'
  });

  // Arrays
  const [todos, setTodos] = useState([
    { id: 1, text: 'Learn React', done: false },
    { id: 2, text: 'Build an app', done: false }
  ]);

  return (<div>{/* Your JSX here */}</div>);
}

Updating State Correctly

State updates can be tricky! Here's how to update different types of state safely.

// ‚úÖ Updating primitives
const [count, setCount] = useState(0);
const increment = () => setCount(count + 1);

// ‚úÖ Updating objects (must spread existing properties)
const [user, setUser] = useState({ name: 'John', age: 30 });
const updateUserName = (newName) => {
  setUser({ ...user, name: newName });
};

// ‚úÖ Updating arrays
const [items, setItems] = useState(['apple', 'banana']);

// Add item
const addItem = (newItem) => {
  setItems([...items, newItem]);
};

// Remove item
const removeItem = (index) => {
  setItems(items.filter((_, i) => i !== index));
};

Functional Updates

When your new state depends on the previous state, use functional updates for reliability!

const [count, setCount] = useState(0);

// ‚ùå Problem: May not work correctly with rapid clicks
const badIncrement = () => {
  setCount(count + 1);
  setCount(count + 1); // Still uses old count value!
};

// ‚úÖ Solution: Use functional update
const goodIncrement = () => {
  setCount(prevCount => prevCount + 1);
  setCount(prevCount => prevCount + 1); // Uses latest value!
};

// Real-world example: Toggle functionality
const [isOpen, setIsOpen] = useState(false);
const toggle = () => setIsOpen(prev => !prev);
Always treat state as immutable! Instead of modifying existing objects/arrays, create new ones with your changes. This helps React detect changes and update the UI correctly.

Part 2: Understanding useEffect Hook üåä

What is useEffect? ü§î

useEffect is a Hook that lets you perform side effects in functional components. Side effects are operations that affect something outside the component - like API calls, timers, subscriptions, or manually changing the DOM.

üîÑ Three Main useEffect Patterns

1. Run Once (Component Mount)

useEffect(() => {
  // Runs once when component mounts
  fetchUserData();
  setupEventListeners();
}, []); // Empty dependency array

Use for: API calls, setting up subscriptions, initializing libraries

2. Run on Specific Changes

useEffect(() => {
  // Runs when count changes
  document.title = `Count: ${count}`;
  saveToLocalStorage(count);
}, [count]); // Dependency array with count

Use for: Responding to specific state/prop changes, syncing with external systems

3. Run Every Render

useEffect(() => {
  // Runs after every render
  console.log('Component updated');
  measureElement();
}); // No dependency array

Use for: DOM measurements, logging (be careful with performance!)

üè† useEffect Like a Smart Home Automation

Think of useEffect like programming a smart home:
‚Ä¢ Empty dependencies [] = "When I move in, turn on the lights" (runs once)
‚Ä¢ With dependencies [temperature] = "When temperature changes, adjust thermostat"
‚Ä¢ Cleanup function = "When I move out, turn off all systems"
‚Ä¢ No dependencies = "Check everything every time I enter a room" (usually too much!)

useEffect Deep Dive - Advanced Patterns üöÄ

Cleanup Functions - Preventing Memory Leaks

Always clean up your side effects to prevent memory leaks and unexpected behavior!

import React, { useState, useEffect } from 'react';

function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    console.log('Setting up timer');
    
    // Setup: Create the timer
    const intervalId = setInterval(() => {
      setSeconds(prev => prev + 1);
    }, 1000);

    // Cleanup: Clear the timer when component unmounts
    return () => {
      console.log('Cleaning up timer');
      clearInterval(intervalId);
    };
  }, []); // Empty dependency - runs once

  return <div>Timer: {seconds} seconds</div>;
}

Dependency Array Rules

Understanding dependencies is crucial for useEffect to work correctly!

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  // ‚úÖ Correct: Include all dependencies
  useEffect(() => {
    fetchUser(userId).then(setUser);
  }, [userId]); // userId is from props, must be in dependencies

  // ‚ùå Wrong: Missing dependencies
  useEffect(() => {
    fetchUser(userId).then(setUser); // Uses userId but not in deps
  }, []); // This will use stale userId!

  // ‚úÖ Correct: Using functional updates to avoid dependencies
  useEffect(() => {
    const timer = setInterval(() => {
      setUser(prevUser => ({ ...prevUser, lastSeen: Date.now() }));
    }, 5000);
    return () => clearInterval(timer);
  }, []); // No dependencies needed with functional update

  return (<div>{/* Your JSX */}</div>);
}

Custom Hooks - Reusing Logic

Extract common useEffect patterns into custom hooks for reusability!

// Custom hook for API calls
function useApi(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let cancelled = false;

    setLoading(true);
    fetch(url)
      .then(response => response.json())
      .then(data => {
        if (!cancelled) {
          setData(data);
          setLoading(false);
        }
      })
      .catch(error => {
        if (!cancelled) {
          setError(error);
          setLoading(false);
        }
      });

    return () => { cancelled = true; }; // Cleanup
  }, [url]);

  return { data, loading, error };
}

// Using custom hooks
function UserDashboard() {
  const { data: users, loading } = useApi('/api/users');

  if (loading) return <div>Loading...</div>;

  return (
    <div>
      {users.map(user => <div key={user.id}>{user.name}</div>)}
    </div>
  );
}

‚ùå Common useEffect Mistakes

// Missing dependencies
useEffect(() => {
  fetchData(userId); // userId not in dependencies
}, []); // ‚ùå Will use stale userId

// Infinite loop
useEffect(() => {
  setData(someObject); // someObject recreated every render
}, [someObject]); // ‚ùå Runs infinitely

// No cleanup
useEffect(() => {
  const timer = setInterval(tick, 1000);
  // ‚ùå Timer never cleared - memory leak!
}, []);

‚úÖ Correct useEffect Patterns

// All dependencies included
useEffect(() => {
  fetchData(userId);
}, [userId]); // ‚úÖ Runs when userId changes

// Stable object with useMemo
const memoizedObject = useMemo(() => ({ key: value }), [value]);
useEffect(() => {
  setData(memoizedObject);
}, [memoizedObject]); // ‚úÖ Only runs when value changes

// Proper cleanup
useEffect(() => {
  const timer = setInterval(tick, 1000);
  return () => clearInterval(timer); // ‚úÖ Cleanup
}, []);

‚ö° Rules of Hooks (VERY IMPORTANT!)

  • Only call Hooks at the top level - Never inside loops, conditions, or nested functions
  • Only call Hooks from React functions - Components or custom hooks
  • Use ESLint plugin - Install eslint-plugin-react-hooks to catch violations
// ‚ùå Wrong: Hook inside condition
if (someCondition) {
  const [state, setState] = useState(0); // DON'T DO THIS!
}

// ‚úÖ Correct: Condition inside hook
const [state, setState] = useState(0);
if (someCondition) {
  setState(newValue); // This is fine
}

Putting It All Together - Complete Example! üéØ

üéØ Real-World Example: User Dashboard with Hooks

Let's build a complete user dashboard that demonstrates useState and useEffect working together!

import React, { useState, useEffect } from 'react';

function UserDashboard() {
  // useState for different types of data
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [searchTerm, setSearchTerm] = useState('');
  const [selectedUser, setSelectedUser] = useState(null);
  const [theme, setTheme] = useState('light');

  // useEffect for initial data loading
  useEffect(() => {
    console.log('Component mounted - fetching users');

    const fetchUsers = async () => {
      try {
        setLoading(true);
        const response = await fetch('https://jsonplaceholder.typicode.com/users');
        const userData = await response.json();
        setUsers(userData);
        setError(null);
      } catch (err) {
        setError('Failed to fetch users');
      } finally {
        setLoading(false);
      }
    };

    fetchUsers();
  }, []); // Empty dependency - runs once on mount

  // useEffect for document title updates
  useEffect(() => {
    document.title = selectedUser
      ? `User: ${selectedUser.name}`
      : 'User Dashboard';
  }, [selectedUser]); // Runs when selectedUser changes

  // useEffect for cleanup example
  useEffect(() => {
    const handleKeyPress = (event) => {
      if (event.key === 'Escape') {
        setSelectedUser(null);
      }
    };

    document.addEventListener('keydown', handleKeyPress);

    // Cleanup function
    return () => {
      document.removeEventListener('keydown', handleKeyPress);
    };
  }, []); // Empty dependency - setup once, cleanup on unmount

  // Event handlers
  const handleSearchChange = (event) => {
    setSearchTerm(event.target.value);
  };

  const handleUserSelect = (user) => {
    setSelectedUser(user);
  };

  const toggleTheme = () => {
    setTheme(prevTheme => prevTheme === 'light' ? 'dark' : 'light');
  };

  // Filtered users based on search
  const filteredUsers = users.filter(user =>
    user.name.toLowerCase().includes(searchTerm.toLowerCase())
  );

  // Render logic
  if (loading) return <div>Loading users...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <div className={`dashboard ${theme}`}>
      <header>
        <h1>User Dashboard</h1>
        <button onClick={toggleTheme}>
          Switch to {theme === 'light' ? 'Dark' : 'Light'} Theme
        </button>
      </header>

      <div className="search-section">
        <input
          type="text"
          placeholder="Search users..."
          value={searchTerm}
          onChange={handleSearchChange}
        />
        <p>Found {filteredUsers.length} users</p>
      </div>

      <div className="users-list">
        {filteredUsers.map(user => (
          <div
            key={user.id}
            className={`user-card ${selectedUser?.id === user.id ? 'selected' : ''}`}
            onClick={() => handleUserSelect(user)}
          >
            <h3>{user.name}</h3>
            <p>{user.email}</p>
            <p>{user.company.name}</p>
          </div>
        ))}
      </div>

      {selectedUser && (
        <div className="user-details">
          <h2>Selected User Details</h2>
          <p><strong>Name:</strong> {selectedUser.name}</p>
          <p><strong>Email:</strong> {selectedUser.email}</p>
          <p><strong>Phone:</strong> {selectedUser.phone}</p>
          <p><strong>Website:</strong> {selectedUser.website}</p>
          <button onClick={() => setSelectedUser(null)}>
            Close Details
          </button>
        </div>
      )}
    </div>
  );
}

export default UserDashboard;

function useCounter(initialValue = 0) {
  const [count, setCount] = useState(initialValue);

  const increment = () => setCount(c => c + 1);
  const decrement = () => setCount(c => c - 1);
  const reset = () => setCount(initialValue);

  return { count, increment, decrement, reset };
}

// Usage
const { count, increment } = useCounter(10);
