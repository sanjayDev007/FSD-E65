JSON vs. Other Formats ü•ä

JSON isn't the only way computers share information! Here's how JSON compares to XML:

JSON - Simple & Light

{
  "pet": {
    "name": "Fluffy",
    "type": "cat",
    "color": "orange"
  }
}

XML - More Complex

<pet>
  <name>Fluffy</name>
  <type>cat</type>
  <color>orange</color>
</pet>

JSON is easier for both humans and computers to read - it has fewer symbols and takes up less space!

Understanding JSON Structure:

  • Property names always have double quotes "name"
  • Property names are followed by a colon and then the value
  • Strings use double quotes "value"
  • Numbers don't need quotes 42
  • Booleans are either true or false
  • Arrays use square brackets [ ] for lists of items
  • Each property is separated by a comma

Real-World JSON üåé

JSON is used EVERYWHERE! Here are some places you might see it:

  • When you play online games - your player data is often stored as JSON
  • Weather apps use JSON to get information about the forecast
  • Social media sites use JSON to share posts and photos
  • When websites remember what's in your shopping cart
  • Maps and navigation apps use JSON for locations and directions
