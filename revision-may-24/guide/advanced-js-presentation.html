1. Functions - Your Code's Best Friends ü§ù

Regular Functions vs Arrow Functions

// Regular Function function sayHello(name) { return "Hello, " + name + "!"; } // Arrow Function (Shorter way!) const sayHelloArrow = (name) => { return "Hello, " + name + "!"; } // Super Short Arrow Function const greet = name => "Hello, " + name + "!"; console.log(sayHello("Alex")); // Hello, Alex! console.log(greet("Sam")); // Hello, Sam!

Think of Functions Like Recipes:

  • Ingredients = Parameters (what you put in)
  • Instructions = Function body (what happens)
  • Final dish = Return value (what you get out)

2. Closures - Functions with Memory üß†

// A function that remembers things! function createCounter() { let count = 0; // This variable is "trapped" inside return function() { count++; // The inner function can access count return count; }; } const counter1 = createCounter(); const counter2 = createCounter(); console.log(counter1()); // 1 console.log(counter1()); // 2 console.log(counter2()); // 1 (separate counter!)

Real-World Example: Bank Account

function createBankAccount(initialBalance) { let balance = initialBalance; return { deposit: (amount) => { balance += amount; return `Deposited $${amount}. Balance: $${balance}`; }, withdraw: (amount) => { if (amount <= balance) { balance -= amount; return `Withdrew $${amount}. Balance: $${balance}`; } return "Insufficient funds!"; }, getBalance: () => balance }; } const myAccount = createBankAccount(100);

3. Callbacks - Functions Calling Functions üìû

// A callback is a function passed to another function function processData(data, callback) { console.log("Processing:", data); // Do some work... callback(data.toUpperCase()); } function displayResult(result) { console.log("Result:", result); } // Pass displayResult as a callback processData("hello world", displayResult); // Output: Processing: hello world // Result: HELLO WORLD

Array Methods with Callbacks

const numbers = [1, 2, 3, 4, 5]; // map - transform each item const doubled = numbers.map(num => num * 2); console.log(doubled); // [2, 4, 6, 8, 10] // filter - keep items that pass a test const evenNumbers = numbers.filter(num => num % 2 === 0); console.log(evenNumbers); // [2, 4] // find - get first item that matches const found = numbers.find(num => num > 3); console.log(found); // 4

Objects - Like Real-World Things

// Object Literal const student = { name: "Emma", age: 15, grades: [85, 92, 78], // Method (function in object) getAverage() { const sum = this.grades.reduce((a, b) => a + b, 0); return sum / this.grades.length; } }; console.log(student.name); // Emma console.log(student.getAverage()); // 85

Classes - Templates for Objects

class Animal { constructor(name, species) { this.name = name; this.species = species; } speak() { return `${this.name} makes a sound!`; } } class Dog extends Animal { constructor(name, breed) { super(name, "Dog"); // Call parent constructor this.breed = breed; } speak() { return `${this.name} barks!`; } wagTail() { return `${this.name} is wagging their tail!`; } } const buddy = new Dog("Buddy", "Golden Retriever"); console.log(buddy.speak()); // Buddy barks!

5. Promises - Dealing with Future Results ‚è∞

Think of Promises like ordering food:

  • Pending: Your order is being prepared
  • Fulfilled: Your food is ready!
  • Rejected: Sorry, we're out of that item
// Creating a Promise function orderFood(item) { return new Promise((resolve, reject) => { console.log(`Preparing ${item}...`); setTimeout(() => { const available = Math.random() > 0.3; // 70% success rate if (available) { resolve(`${item} is ready! üçï`); } else { reject(`Sorry, ${item} is out of stock üòû`); } }, 2000); // Wait 2 seconds }); } // Using the Promise orderFood("Pizza") .then(result => console.log(result)) .catch(error => console.log(error));

Async/Await - Cleaner Way to Handle Promises

async function getFood() { try { const result = await orderFood("Burger"); console.log(result); console.log("Enjoy your meal!"); } catch (error) { console.log(error); console.log("Maybe try something else?"); } } getFood();

7. Error Handling - When Things Go Wrong üö®

// Try-Catch - Like a safety net function divideNumbers(a, b) { try { if (b === 0) { throw new Error("Cannot divide by zero!"); } return a / b; } catch (error) { console.log("Oops! Something went wrong:", error.message); return null; } finally { console.log("Division operation completed"); } } console.log(divideNumbers(10, 2)); // 5 console.log(divideNumbers(10, 0)); // null (with error message)

Common Error Types:

  • SyntaxError: Typos in your code
  • TypeError: Using wrong data type
  • ReferenceError: Using undefined variables
  • RangeError: Number out of range
// Async error handling async function fetchUserData(userId) { try { const response = await fetch(`/api/users/${userId}`); if (!response.ok) { throw new Error(`User not found: ${response.status}`); } const user = await response.json(); return user; } catch (error) { console.log("Error fetching user:", error.message); return { error: "User not found" }; } }

8. Modern JavaScript Features üåü

Destructuring - Unpacking Values

// Array Destructuring const colors = ["red", "green", "blue"]; const [first, second, third] = colors; console.log(first); // red // Object Destructuring const person = { name: "Alice", age: 15, city: "New York" }; const { name, age } = person; console.log(name); // Alice // Function parameter destructuring function greetPerson({ name, age }) { return `Hello ${name}, you are ${age} years old!`; } console.log(greetPerson(person));

Template Literals - Better Strings

const name = "JavaScript"; const year = 2024; // Old way const message1 = "I love " + name + " in " + year + "!"; // New way (template literals) const message2 = `I love ${name} in ${year}!`; // Multi-line strings const poem = ` Roses are red, Violets are blue, JavaScript is awesome, And so are you! `;

Spread Operator - Expanding Things

// Array spreading const fruits = ["apple", "banana"]; const moreFruits = ["orange", "grape"]; const allFruits = [...fruits, ...moreFruits]; console.log(allFruits); // ["apple", "banana", "orange", "grape"] // Object spreading const basicInfo = { name: "John", age: 15 }; const fullInfo = { ...basicInfo, school: "High School", grade: 10 }; console.log(fullInfo);
